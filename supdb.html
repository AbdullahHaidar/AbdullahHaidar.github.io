Supermarket Database Management

Microsoft Sql Server 2014 Project

Github Directory


The database contains all the data from the supermarket: customers, employees, products, stocks ,orders …..

The sql server used is Microsoft Sql Server 2014, with authentication mode.

Below is a breif specification of the main features implemented in the application:

Procedures:

Data Procedures
  1. Procedure to add Customers.
  2. Procedure to add Employees.
  3. Procedure to add Products.
  4. Procedure to add Orders: the addOrder procedure only takes two arguments: the Customer Id and the Order Id , the Page 19 date and name of id of the employee is taken from GetDate() and @@CURRENT_USER.
  5. Procedure to add an item to an order.
  6. Procedure to add quantity to stocks.
Security Procedures
  1. Procedure to create Users.
  2. Procedures to grant permissions to users.
Statistics Procedures
  1. Procedure to get the amount gained with respect to days, months, years….
  2. Procedure to get the average of sales with respect to days, months, years….
  3. Procedure to get the nb of orders with respect to days, months, years

Triggers:

  1. Foreign key triggers for all the tables, materializing referential constraints.
  2. Trigger checkStock to make sure a product is not included in an order if its quantity becomes lower then the allowed minimum stock quantity.

Views:

We implement some views for security, every employee has a view of his personal information, and he is denied access to others employees information.
We also implements some other useful views:
  1. OrdersView
  2. ) Order_Products_ExtraInformation
  3. Order_Products

Security:

  1. Create logins and users for the employees.
  2. Create roles Cashier and StockManager, and grant each one the relevant permissions.
  3. Grant the convenient role to suitable user.
